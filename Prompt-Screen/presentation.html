Key Concepts

Core Modules

The engine is built from composable, single-responsibility modules.

Dual-Mode Routing

Known prompts use deterministic domain knowledge. Unknown prompts call Ollama AI for dynamic generation.

Component Mapping

Resolved fields are mapped to UI components (tables, cards, forms) with layout semantics built in.

JSON Output

Final output is render-ready JSON frames compatible with Figma rendering and design-system viewers.

File Map

Project Structure

Each module has a single responsibility in the generation pipeline.

aiuxengine.js Main entry point — CLI and Express API server
orchestrator/ appOrchestrator.js — coordinates the full generation pipeline
engine/ domainResolver, runModeResolver, fieldResolver, componentMapper
ai/ flowGenerator, itemNameResolver, contextResolver — Ollama AI layer
builder/ layoutBuilder, jsonBuilder — convert resolved data into JSON frames
renderer/ figmaRenderer.js — produces Figma-compatible output
config/ domainModels.json — domain knowledge catalog (entities, fields, prompts)
knowledge/ Extended domain knowledge, flow templates, design tokens, rules
render-output.json Generated output — array of Figma-compatible frames

Knowledge Base

Domain Model Example

A compact student model drives fields, labels, and CTA text deterministically.

{ "entity": "student", "fields": [ { "name": "name", "type": "text" }, { "name": "email", "type": "email" } ], "cta": "Create Student" }
Field Resolver extracts name and email from the domain model to build form inputs automatically.
Component Mapper selects the right input type and validation rules for each field.
CTA text stays deterministic — always derived from the model, never guessed by AI.

Architecture Philosophy

AI vs Knowledge

Deterministic knowledge first, AI only as a fallback when domain rules are insufficient.

Primary
Knowledge-Driven
  • Flow selection based on domain rules — no guessing
  • Fields and CTA text locked to the domain model
  • Predictable, repeatable layouts via the slot system
  • Fully explainable — every decision is traceable
Fallback
AI-Assisted
  • Intent enrichment when the domain is unclear
  • Copy suggestions when labels are missing
  • Never overrides knowledge rules or slot structure
  • Always flagged and auditable in the output
